1. What Gets Built in MVP 
The MVP delivers a full working platform capable of onboarding, training, and certifying real 
paying learners at small enterprise scale. It is built across seven modules in priority order. 
 
Module 1 — AI Tutor Engine (P0) 
This is the heart of the product. Everything else supports it. 
• 
RAG pipeline: course content chunked and stored as vector embeddings in Pinecone; AI 
pulls relevant chunks before every response 
• 
Conversational session management with full conversation history retained per session 
• 
Incorrect answer handling: AI explains why the answer is wrong, provides an alternative 
explanation, then re-tests the concept 
• 
Five-condition escalation trigger logic (see Architecture section) 
• 
Session summary generated at end of each session: what was covered, understood, 
needs reinforcement, and next steps 
• 
Response latency target: under 2.5 seconds for 95% of interactions 
• 
Full session transcript stored per session for QA and human tutor review 
 
Module 2 — Course Content Management System (P0) 
• 
Drag-and-drop course builder for tutors with no technical knowledge required 
• 
Content types: text lessons, video (upload or linked), PDF downloads, quizzes, 
assessments, case studies 
• 
Course version control with rollback to any previous version 
• 
AI-assist first draft generation for lessons — tutors review and approve before publishing 
• 
Content tagging: skill area, domain, difficulty, learning objective 
• 
Publish / unpublish workflow with staging preview 
• 
10 initial MVP courses pre-loaded across Finance, Technology, Leadership, and Data 
Analytics 
 
Module 3 — Personalized Learning Path Engine v1 (P1) 
• 
10-minute onboarding diagnostic combining background questions and a short skills 
assessment 
• 
Rule-based path recommendation: compares learner current skill profile against their 
career goal 
• 
Sequenced course path with milestone markers 
• 
Weekly progress report: completed, on track, upcoming 
• 
Path acceleration when mastery is faster than expected; reinforcement added when 
learner struggles

Module 4 — Assessment and Certification Engine (P0) 
• 
Inline knowledge checks embedded in lessons — unscored, for recall reinforcement 
• 
Module assessments — scored, results stored in learner profile 
• 
Final course examination — minimum pass score configurable per course 
• 
Practical assignment submissions routed to human tutor review queue 
• 
Auto-generated PDF certificate with unique verification code upon passing 
• 
Public certificate verification page — no authentication required 
• 
One-click LinkedIn share from certificate page 
 
Module 5 — Enterprise Administration Portal (P0) 
• 
Bulk CSV user import — auto-creates accounts and sends invite emails 
• 
Department and team grouping and management 
• 
Course assignment to individuals, teams, or entire organisation with deadlines 
• 
Real-time dashboard: completion percentage, assessment scores, at-risk learners, daily 
active users 
• 
Automated WhatsApp and email nudges for inactive or deadline-approaching learners 
• 
PDF and Excel compliance report export in under 30 seconds 
• 
Budget tracking: spend per learner against allocation 
• 
SAML 2.0 enterprise SSO — clients like KPMG and Access Bank use their existing 
identity providers 
 
Module 6 — Human Tutor Integration Layer (P1) 
• 
Escalation queue with full transcript viewer and annotation tools 
• 
Embedded video response recorder for tutors 
• 
Live 1:1 session scheduler with calendar integration 
• 
Knowledge base contribution interface — tutors submit corrections to AI knowledge base 
• 
Tutor earnings dashboard: cases resolved, hours, payment due 
 
Module 7 — Notification and Engagement Engine (P1) 
• 
Email via Resend: certificates, reports, and formal platform updates 
• 
WhatsApp via Twilio: nudges, reminders, and quick check-ins (primary Nigerian channel) 
• 
In-app notifications: real-time updates while on platform 
• 
SMS fallback: for learners with limited data access 
• 
Intelligent triggers: 3-day inactivity, score drop, deadline approaching, course completion

3. Technical Architecture 
Architecture Pattern 
Elite Coach AI is built as a modular monolith with clean service boundaries, enabling 
straightforward decomposition into microservices when traffic demands it. Each business 
domain (auth, tutor, courses, enterprise, payments) is a self-contained module with its own 
controller, service, and repository layer. 
 
RAG Pipeline — Step by Step 
Step 
Name 
Description 
1 
Content Ingestion 
Tutor publishes course content via CMS — split into concept-level 
chunks 
2 
Embedding 
Generation 
Each chunk sent to OpenAI Embeddings API — produces dense vector 
representation 
3 
Vector Storage 
Embeddings stored in Pinecone, indexed by course_id, module_id, and 
topic 
4 
Learner Query 
Learner message converted to a vector embedding at session time 
5 
Similarity Search 
Top 5 semantically similar content chunks retrieved from Pinecone 
6 
Prompt Assembly 
Retrieved chunks + conversation history + learner question combined 
into a prompt 
7 
LLM Call 
Prompt sent to GPT-4o or Claude 3.5 Sonnet via API 
8 
Response Delivery 
Grounded response streamed to learner via SSE in under 2.5 seconds 
 
Escalation Trigger Conditions 
The AI automatically escalates a session to a human tutor when any of the following are true: 
• 
Learner has asked the same question 3 or more times without answering correctly on 
the subsequent knowledge check 
• 
Sentiment classifier detects significant frustration language in the learner's messages 
• 
The course module is flagged as human-required by the content team 
• 
Pinecone similarity score is below the minimum threshold — AI cannot find relevant 
content to answer the question 
• 
Session exceeds 90 minutes without a break — wellbeing check triggers and AI 
suggests the learner pause
